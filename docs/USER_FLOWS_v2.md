## F0: 初回起動フロー

```
アプリ起動 → [未認証?] → S1 ログイン → Supabase Auth
  → [初回?] → S2 オンボーディング
    → ペルソナ選択（何に変えたい？）
      └ 事業 / 作品 / 仕事 / 知の探求 から複数選択可
    → S3 ホーム（メモ0件 = エンプティステート）
  → [既存] → S3 ホーム
```

### エンプティステート（メモ0件）
- 幾何学モチーフ（同心円 + 十字線）を画面中央に
- テキストは「0 nodes」のみ（極小、mono）
- 録音ボタンが視覚的に唯一のアクション候補

---

## F1: 録音→外部知識紐づけフロー

```
S3 ホーム → タブバー中央(◉)タップ → S4 録音画面

S4 録音画面:
  [idle] 画面中央に録音ボタン（◉）
    → タップで録音開始
  [recording] 波形表示 + タイマー + 進捗リング
    → タップ or 60秒で停止
  [processing] 段階的SSE表示:
    1. transcript 表示（ラベルなし、テキストのみ）
    2. 構造化表示（summary + keyword tags）
    3. 外部知識カード表示（タイムライン構造で3件）
    4. 完了
  [done] ← アイコンタップで S3 ホームに戻る
```

### 外部知識カードの構成
```
○ latent_question（italic、問いの提示）
│
├● 見出し1
│  事実説明（2-3文、品のある端的なトーン）
│  ソースリンク ↗        ♡
│
├● 見出し2
│  事実説明
│  ソースリンク ↗        ♡
│
└● 見出し3
   事実説明
   ソースリンク ↗        ♡
```

### エッジケース
- マイク権限拒否 → 権限要求UIを表示
- 録音0秒 → 何も起こらない（エラーなし）
- API失敗 → 「再試行」アイコンボタン + テキスト「通信エラー」
- メモ上限到達 → モーダルで上限通知

## F2: ホーム画面（メモ一覧）

```
S3 ホーム:
  ヘッダー: [幾何学ロゴ]  [メモ数 / mono]  [≡ メニュー]
  
  メモリスト（新しい順）:
    各行: [summary]  [●●● 外部知識ドット数]
          [相対時間]
    → タップで S5 メモ詳細
  
  ヘッダーメニュー(≡):
    → S8 フォルダ一覧
    → S10 設定
```

### メモカードのドット
- メモに紐づく外部知識の数を小さなドットで表示
- ブックマーク済みの知識は濃いドット、未保存は薄いドット
- 数が視覚的に分かるだけ。タップ対象ではない

---

## F3: メモ詳細

```
S5 メモ詳細 /memo/[id]:
  ヘッダー: [← 戻る]
  
  セクション1: transcript（原文）
  セクション2: summary + keywords
  セクション3: 外部知識一覧（F1で生成されたもの）
    → 各知識カード: 見出し + 事実 + リンク + ♡
  
  下部アクション:
    [フォルダに移動]  [削除]  [コピー]
```

### フォルダ移動
- タップでフォルダ選択のリストが展開
- 既存フォルダ or 新規作成
- domain自動分類は残すが、手動移動も可能

---

## F4: グラフ画面（ノード＝コア体験）

```
S6 グラフ /graph:
  ヘッダー: [all / 7d / 30d フィルタ]  [n nodes · m links]
  
  メイン: D3 force graph（フルスクリーン）
    ノード = メモ（circle + 短縮テキスト）
    エッジ = 既存接続（solid thin line）
  
  タブバー: 通常表示
```

### インタラクション1: ノードタップ（情報閲覧）
```
ノードタップ
  → 選択ノードのstroke太く (#222)
  → 非関連ノード opacity 0.3
  → 関連ノード（接続済み）stroke #999
  → グラフ下部にミニ詳細パネル:
      [summary / 1行]
      [キーワード tags]
      [→ 詳細]  [⊕ 展開]
```

### インタラクション2: 外部知識の展開（⊕ ボタン）
```
ミニパネルの [⊕ 展開] タップ
  → 選択ノードの周囲に外部知識が小ノードとして放射状に出現
  → 外部知識ノード: 小circle / stroke dashed #CCC / テキスト = 見出しの先頭数文字
  → メモノード ↔ 外部知識ノード を thin dashed line で接続
  → 外部知識ノードタップ → 下部パネルに見出し+事実+リンク
  → もう一度 [⊕] or 別ノードタップで外部知識を畳む
```

**これが「外部アイデアを押し込みで表示」の実装。** グラフ空間内に外部知識が広がって出てくる。

### インタラクション3: 掛け合わせ（アイデア生成）
```
ノードAタップ（選択状態）
  → ノードBタップ
  → A-B間に dashed line がアニメーション描画（400ms）
  → ローディング（線が pulse）
  → 結果カードがグラフ下部にスライドイン:
      [● A名] x [● B名]
      [掛け合わせテキスト / 構造指摘 / 品のあるトーン]
      [♡]  [deep dive →]
  → グラフ上のA-B間の線が solid に変わる（接続確定）
```

### インタラクション4: 紐付け（簡易接続）
```
ノードAを長押し（500ms）
  → ドラッグモード開始（ノードAから指に線が伸びる）
  → ノードBの上でドロップ
  → 確認なしで即座に接続線を追加（軽量。掛け合わせAI不要）
  → 後からその線をタップすると掛け合わせを走らせることも可能
```

### エンプティステート（メモ0-1件）
- 0件: 同心円モチーフ + 「0 nodes」
- 1件: ノード1つだけ画面中央に。接続線なし
- 2件以上: force simulation開始

### エッジケース
- 同一ノード2回タップ → メモ詳細への遷移（S5）
- 掛け合わせAPI失敗 → dashed lineが赤くフラッシュ → 消える
- ノード数が多い（20+） → ピンチズームで対応。クラスタリングはv2

## ブックマーク（♡）のデータフロー

```
外部知識カード上で ♡ タップ
  → connections テーブルの bookmark フィールドを true に
  → ホームのメモカードのドット色が濃くなる
  → メモ詳細でブックマーク済み知識にマーク表示
  → グラフで外部知識展開時、ブックマーク済みは stroke が濃い
```

---

## 「もっと見る」（外部知識の追加生成）

```
録音結果画面で3件表示後
  → 最下部に「more」テキストリンク
  → タップで追加3件を生成（別のlatent_question or 別角度）
  → 既存3件の下に追加表示
  → 1回のみ（合計最大6件）
```

---

## パフォーマンス考慮

- 録音結果: SSEで段階表示（体感速度向上）
- グラフ: 20ノード以下は即時描画。20以上は初期表示を中心5ノードに絞り、ズームアウトで全体表示
- 外部知識展開: 既にDBにあるデータを表示するだけ（API不要）
- 掛け合わせ: API呼び出し。ローディングは線のpulseで表現
